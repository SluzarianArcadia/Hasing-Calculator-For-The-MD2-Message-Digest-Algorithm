MD2 Hash Tutorial Website

Welcome to my tutorial calculator for the MD2 hashing algorithm. MD2 is an older hasing algorithm and a nice starting place to learn how hashing works. The simplest explanation of hashing is that you take a string of characters and move it through an algorithm to get another set of characters.

Why Use A Hashing Algorithm

  • Data Structure to Quickly Look Data Up
  • Verifying the Integrity of Messages and Files
  • Secure Password Storage and Verification
  • Storing Passwords as Hash Digests Instead of Plaintext
  • Signature Generation and Verification
  • Proof-of-work

More specifically hashing algorithms can take any data of any size (even "") and turn it into a fixed size of values. That could be your password, a whole text of a book, the zero and ones of a file, and any picture.

Hashing algorithms are a one way process unlike key algorithms. It's mathematical unlikely a hacker will form a reverse algorithm to turn all the hashes back to plain-text. This would mean the algorithm has been cracked (which this one has been so please donâ€™t use it!).
